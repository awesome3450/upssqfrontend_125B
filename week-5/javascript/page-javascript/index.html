Script Loading Strategies

All the html in the page is loaded in the order in which its appears, if u are using javascript to manipulate elements on the page(or more accuratetly in the DOM) your code won't work if the javascript is loaded and parsed before the html u are trying to do something to.


There a few diffrerent startegies to make sure your javascript only runs after the html is parsed.

  • In the extetrnal javascript above which you deleted, the script element is placed at the bottom of the body of the element and then only runs after the rest of the html body is parsed.
  • In the external javascript above, the script element is placed in the head above before the html body is parsed. But because we are using type="module", the code is treated as the module and the browser wait for all html to be processed before executing javascript module (you could also placed external script at the bottom of the body, but if the html is long and the network is slow, it may take time before the broswr can even start fetching and loading the script, so placing external script in the head is usually better.)
  • If u styill want to use non-module script in the document head, which could block the whole page from displaying and which could cause errors
  • For external script, we should add the defer(or if u don't need the html to be ready they are synced attribute on the script element)
  • For internal script, you should wrap the code in the domContentLoadedEventListener.
